# 저장소 안내

이 저장소는 SK이노베이션 환경분석팀의 일일 시황 스크리닝(`news-log/`) 및 주간 리포트(`weekly-topics/`) 산출물을 관리한다.

## 일일 시황 스크리닝 루틴 — 날짜 확정 절차 (필수)

매일 08:30 KST 트리거로 실행되는 "일일 시황 스크리닝" 작업을 수행할 때는, 뉴스 검색이나 레포 상태 확인보다 **먼저** 아래 절차로 "오늘" 날짜를 확정한다.

1. `mcp__PlayMCP__NaverSearch-get_current_korean_time` 도구를 호출해 현재 KST 날짜를 직접 확인한다. (세션 시스템 컨텍스트에 `currentDate`가 주어져 있어도, 그 값과 실제 KST 시각이 일치하는지 이 도구로 교차검증한다.)
2. **`news-log/` 폴더에서 가장 최근 파일(ls로 정렬했을 때 마지막 파일)을 "오늘"로 착각하지 않는다.** 가장 최근 파일은 어제 또는 그 이전에 정상적으로 생성된 결과물일 수 있다 — "파일이 존재한다"는 사실이 "그 파일 날짜 = 오늘"을 의미하지 않는다.
3. 1번에서 확정한 오늘 날짜(YYYY-MM-DD, KST 기준)로 `news-log/YYYY-MM-DD.md` 경로가 **정확히 그 날짜로** 이미 존재하는지 확인한다.
   - 존재하고 내용이 정상이며 이메일도 이미 발송(Sent) 확인되면 → 중복 실행이므로 추가 작업 없이 종료.
   - 존재하지 않으면 → 정상적으로 오늘자 작업(뉴스 수집 → 분석 → 파일 저장 → 이메일 → main 병합)을 처음부터 진행.

이 순서를 어기고 "레포에 있는 최신 파일 = 오늘"로 추론하면, 실제로는 어제 날짜인데 오늘 작업을 건너뛰는 오류가 발생한다 (2026-07-16 실행 시 실제 발생한 사례).

## 주간 리포트 루틴 — 날짜 확정 절차 (필수)

매주 월요일에 실행되는 "주간 시황 주제 추천" 작업(`weekly-topics/YYYY-MM-DD.md` 생성)도 일일 스크리닝과 동일한 날짜 착오 위험이 있다. 작업 시작 전 아래 절차를 반드시 따른다.

1. `mcp__PlayMCP__NaverSearch-get_current_korean_time` 도구로 현재 KST 날짜를 직접 확인한다. 세션 시스템 컨텍스트의 `currentDate`가 있어도 이 도구로 교차검증한다.
2. **`weekly-topics/` 폴더의 가장 최근 파일을 "이번 주 월요일"로 착각하지 않는다.** 예를 들어 오늘이 2026-07-20(월)인데 폴더에 `2026-07-13.md`만 있다고 해서 "지난주 리포트가 이번 주 것"이라고 오인하거나, 반대로 "이미 이번 주 리포트가 있다"고 오인해서는 안 된다 — 파일명의 날짜와 1번에서 확인한 오늘 날짜를 문자 그대로 비교해서 판단한다.
3. 1번에서 확정한 오늘 날짜가 속한 주의 월요일 날짜(YYYY-MM-DD)를 계산하고, `weekly-topics/YYYY-MM-DD.md`가 **정확히 그 날짜로** 이미 존재하는지 확인한다.
   - 존재하면 → 중복 실행이므로 추가 작업 없이 종료.
   - 존재하지 않으면 → 정상적으로 작업(지난 7일치 `news-log/` 로그 종합 → 주간 리포트 작성 → 커밋)을 진행.
4. 지침상 "지난 7일치 데일리 로그"를 종합해야 하는 경우, 그 7일 구간도 1번에서 확정한 오늘 날짜를 기준으로 역산한다 (레포에 있는 파일 목록을 보고 임의로 구간을 추정하지 않는다). `news-log/` 에 7일치가 모두 없으면 있는 파일만 사용하되, 몇 일치 데이터로 작성했는지 리포트 상단에 명시한다 (2026-07-13 리포트에서 이미 사용한 방식).

날짜를 잘못 확정하면 지난주 리포트를 이번 주 것으로 오인해 작업을 건너뛰거나, 반대로 이미 존재하는 리포트를 중복 생성하거나, 종합 대상 7일 구간 자체가 틀어질 수 있다.

## Gmail 발송 관련 참고

이 세션에 연결된 Gmail MCP 도구는 `create_draft`만 제공하며 실제 발송(send) 기능이 없다. 초안 생성 후 자동으로 수신자에게 전달되지 않으므로, 이메일 발송이 루틴에 포함된 작업에서는 초안 생성 후 반드시 "초안만 생성되고 자동 발송되지 않았다"는 점을 세션 요약과 알림에 명시한다.
