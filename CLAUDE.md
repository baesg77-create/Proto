# 저장소 안내

이 저장소는 SK이노베이션 환경분석팀의 일일 시황 스크리닝(`news-log/`) 및 주간 리포트(`weekly-topics/`) 산출물을 관리한다.

## 일일 시황 스크리닝 루틴 — 날짜 확정 절차 (필수)

매일 08:30 KST 트리거로 실행되는 "일일 시황 스크리닝" 작업을 수행할 때는, 뉴스 검색이나 레포 상태 확인보다 **먼저** 아래 절차로 "오늘" 날짜를 확정한다.

1. `mcp__PlayMCP__NaverSearch-get_current_korean_time` 도구를 호출해 현재 KST 날짜를 직접 확인한다. (세션 시스템 컨텍스트에 `currentDate`가 주어져 있어도, 그 값과 실제 KST 시각이 일치하는지 이 도구로 교차검증한다.)
2. **`news-log/` 폴더에서 가장 최근 파일(ls로 정렬했을 때 마지막 파일)을 "오늘"로 착각하지 않는다.** 가장 최근 파일은 어제 또는 그 이전에 정상적으로 생성된 결과물일 수 있다 — "파일이 존재한다"는 사실이 "그 파일 날짜 = 오늘"을 의미하지 않는다.
3. 1번에서 확정한 오늘 날짜(YYYY-MM-DD, KST 기준)로 `news-log/YYYY-MM-DD.md` 경로가 **정확히 그 날짜로** 이미 존재하는지 확인한다.
   - 존재하고 내용이 정상이며 이메일도 이미 발송(Sent) 확인되면 → 중복 실행이므로 추가 작업 없이 종료.
   - 존재하지 않으면 → 정상적으로 오늘자 작업(뉴스 수집 → 분석 → 파일 저장 → 이메일 → main 병합)을 처음부터 진행.

이 순서를 어기고 "레포에 있는 최신 파일 = 오늘"로 추론하면, 실제로는 어제 날짜인데 오늘 작업을 건너뛰는 오류가 발생한다 (2026-07-16 실행 시 실제 발생한 사례).

## Gmail 발송 관련 참고

이 세션에 연결된 Gmail MCP 도구는 `create_draft`만 제공하며 실제 발송(send) 기능이 없다. 초안 생성 후 자동으로 수신자에게 전달되지 않으므로, 이메일 발송이 루틴에 포함된 작업에서는 초안 생성 후 반드시 "초안만 생성되고 자동 발송되지 않았다"는 점을 세션 요약과 알림에 명시한다.
